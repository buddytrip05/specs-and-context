# AI-Powered PM Workshop
### *The Checkout Abandonment Mystery*

> "You don't need better prompts. You need better systems."

---

## What this is

A hands-on workshop for working product managers who want to use AI — specifically Claude — to do their actual job faster and better. Not theory. Not demos. We build real artifacts together, in real time, and you leave with a workflow you can use Monday morning.

**The problem we solve together:** Terracart, a fictional mid-market e-commerce platform, is losing $2.1M/month at the payment step. Nobody agrees on why. We fix it — with AI.

---

## The five builds

1. **Messy notes → PRD draft** — Start with chaos, end with structure
2. **Interview transcript → evidence-backed insight memo** — Synthesis, not summaries
3. **One source → three audiences** — Exec brief, engineering handoff, Slack update
4. **Build your context folder** — The system that makes every output better
5. **PRD reviewer workflow** — Generate, then audit

---

## The core lesson

> Output quality improves when PMs improve the *system* around the model — not just the prompt.

Every exercise is designed to prove that claim by showing the before/after in real time.

---

## How to use this repo

- **During the workshop:** Follow the exercise guides in `03-exercises/`. Raw inputs are in `02-raw-inputs/`.
- **Prompts are in `04-prompts/`** — copy, paste, modify. They're yours.
- **After the workshop:** The context folder structure in `03-exercises/04-context-folder.md` is the thing to actually implement at work.
